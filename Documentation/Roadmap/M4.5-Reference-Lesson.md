# M4.5 Reference Lesson

## Goal

Promote hand-authored lessons into permanent benchmark lessons for ASM-Lab.

## Purpose

After the minimal executor passes a golden fixture:

- the lesson becomes a reference acceptance test
- future executor and explanation changes must continue to satisfy it
- regressions in memory intuition are caught immediately

Current reference lessons:

1. `Stack Strings - calc.exe`
2. `Pointer Dereference`
3. `Stack Frame - Canonical x86 Prologue`

## Required Validation

For every affected step in a reference lesson, the project must validate:

- fixture validation
- trace validation
- state diff validation
- inspect validation

These checks should remain separate where appropriate:

- semantic fixture comparison for executor, diff, and explanation artifacts
- inspect comparison for inspect-view rendering artifacts

## Rule

If generated executor or explanation output diverges from the hand-authored fixture:

- fix the executor
- fix the explanation engine
- or refine the models

Do not add new instruction support until the reference lesson remains stable.

No instruction support may be added until a lesson fixture requires it.
