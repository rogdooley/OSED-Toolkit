# Raw Layout Builder

`Tools.rawlayout` builds a byte array from a JSON/YAML spec.

Think of it as:
1. Maintain a cursor (`where append writes next`).
2. Execute each segment in order.
3. Write bytes either at cursor (`append`) or at explicit offset (`at`).
4. Reject accidental overlaps unless explicitly allowed.
5. Emit final bytes + a layout report.

## What `append` Means

Your snippet:

```json
{
  "append": {
    "name": "header",
    "source": "text",
    "value": "PING "
  }
}
```

Meaning in plain terms:
- `append`: write at the current cursor.
- `name`: label for reporting/debug output.
- `source: "text"`: turn the string into bytes (UTF-8 by default).
- `value`: actual text to convert.
- After writing, cursor moves forward by `len(value)` bytes.

So if cursor starts at `0`, this writes 5 bytes (`50 49 4E 47 20`) at `[0,5)`.

## Layout Ops (Core)

- `label`: save current cursor as a named offset.
- `append`: write bytes at cursor, then advance cursor.
- `pad_to`: advance cursor to target offset by filling gaps.
- `at`: write bytes at an exact offset (cursor does not move unless `advance_cursor: true`).
- `assert_offset`: fail if cursor is not exactly the expected offset.
- `assert_max_size`: fail after assembly if final payload size exceeds the value.

## Top-Level Fields

- `params`: named numeric inputs (`eip_offset`, `max_len`, etc.).
- `fill_byte`: default byte for unwritten gaps when materializing final payload.
- `external_commands`: named command arrays for `source: "external"`.
- `external_validation`: default validators for external-generated bytes.
- `constraints`: common final constraints (`assert_len_max`, `assert_len_exact`).
- `final_validation`: additional final validators (`badchars`, mutator checks, etc.).
- `segments`: ordered list of layout operations.

## Segment Source Types

- `text`: encode a string (`value`) to bytes.
- `bytes`: escaped bytes string (`"\\x41\\x42"`).
- `hex`: hex string (`"414243"`).
- `int`: pack integer (`value`, `size`, `endian`, `signed`).
- `file`: read bytes from file (`path`).
- `external`: run command by `command_ref` and capture stdout bytes.
- `computed`: call a registered Python function with `args` and `BuildContext`.

`BuildContext` fields passed to computed functions:
- `current_offset`
- `segment_offsets` (`name -> (start, end)`)
- `total_size_so_far`

## Expressions

Usable in offsets/targets/constraints:
- literal: `64` or `"0x40"`
- param: `{ "param": "marker_offset" }`
- label ref: `{ "ref": "after_header" }`
- add: `{ "add": [expr_a, expr_b] }`
- sub: `{ "sub": [expr_a, expr_b] }`

## Minimal Example

File: `Tools/rawlayout/examples/minimal_append.json`

```json
{
  "segments": [
    { "append": { "name": "hello", "source": "text", "value": "HELLO" } }
  ]
}
```

## Offset + Marker Example

File: `Tools/rawlayout/examples/basic_layout.json`

Highlights:
- `params.marker_offset` controls where marker is placed.
- `pad_to` reaches marker offset.
- `at` writes marker value at that absolute offset.
- second `pad_to` advances past marker.
- `append tail` continues right after marker.

## External Source Example

File: `Tools/rawlayout/examples/external_source.json`

Highlights:
- defines named external command in `external_commands`.
- segment with `source: "external"` pulls bytes from that command.
- validates external bytes length.

## Assertion Example

```json
{
  "segments": [
    { "append": { "name": "hdr", "source": "text", "value": "HDR" } },
    { "assert_offset": 3 },
    { "append": { "name": "body", "source": "bytes", "value": "\\x41\\x42" } },
    { "assert_max_size": 32 }
  ]
}
```

## CLI

```bash
python -m Tools.rawlayout.cli.build_payload \
  --spec Tools/rawlayout/examples/basic_layout.json \
  --dump-bytes /tmp/out.bin \
  --print-layout-report
```

Useful flags:
- `--param key=value` override `params`.
- `--external-command name='["argv0","arg1"]'` override command refs.
- `--external-cmd-json '{...}'` bulk command overrides.
- `--dump-bytes FILE` write output bytes.
- `--print-layout-report` print ranges/labels/overlaps.
- `--layout-report-format table|json` choose report format.
- `--verbose` print SHA-256 of built bytes.

## Naming and Migration (from `Tools/raw`)

| Old (`Tools/raw`) | New (`Tools/rawlayout`) |
|---|---|
| `until_offset` / `pad_to_offset` | `pad_to.target` |
| `at_offset` | `at.offset` |
| `dword` + `endian` | `source: "int"` + `value` + `size` + `endian` |
| `value` + `width` | `source: "int"` + `value` + `size` |
| `bytes_file` | `source: "file"` + `path` |
| `raw_bytes` | `source: "bytes"` or `source: "hex"` |
| `badchars` (root) | `final_validation.badchars` |
| `expected_total_size` | `constraints.assert_len_exact` |

## External Command Safety

`generate_external_bytes()` uses:
- explicit argv list (no `shell=True`)
- timeout
- max output bytes
- raw stdout bytes (no text transcoding)
